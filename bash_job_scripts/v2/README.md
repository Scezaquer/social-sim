# V2 experiment suite

All V2 runs are rows of committed design CSVs (`designs/*.csv`), generated by
`generate_design.py` (deterministic, seed 20260612). This replaces V1's on-node
`$RANDOM` sampling: marginals are exactly balanced, model x factor cells are
exactly balanced, and every run is reproducible from its design row (the row's
`seed` seeds python/numpy/torch inside `src/main.py`).

See `cookbook_rebuttal/V2_PLAN.md` for the reviewer-issue → experiment mapping.

## Experiments

| Script | Design | Runs | What it answers |
|---|---|---|---|
| `mila_v2_core_sweep.sh` | `e1_core_sweep.csv` | 576 | Main design-space sweep (model, proportions, question, N, graph, homophily, ctx, news dose, activity exponent) |
| `mila_v2_base_vs_lora.sh` | `e2_base_vs_lora.csv` | 144 | No-LoRA arm for all 4 base models → crossed model x fine-tuning analysis |
| `mila_v2_topology.sh` | `e3_topology.csv` | 336 | First-class topology study, 7 density-matched topologies x 4 models x 3 questions |
| `mila_v2_noise_floor.sh` | `e4_noise_floor.csv` | 96 | Survey-measurement noise floors (none / scrambled stimulus) |
| `mila_v2_scale_cadence.sh` | `e5_scale_cadence.csv` | 72 | Scale with per-agent-matched survey cadence (kills the V1 OSR artifact) |

Total: 1,224 runs. If compute-constrained, cut E5 first, then E3's non-Minitaur
models, then regenerate E1 with fewer runs; do NOT cut E2 or E4 (they answer the
score-blocking reviews).

## How to run

```bash
# 0) (only if designs change) regenerate the design CSVs and commit them
python bash_job_scripts/v2/generate_design.py

# 1) from the repo root, on the cluster
sbatch bash_job_scripts/v2/mila_v2_core_sweep.sh
sbatch bash_job_scripts/v2/mila_v2_base_vs_lora.sh
sbatch bash_job_scripts/v2/mila_v2_topology.sh
sbatch bash_job_scripts/v2/mila_v2_scale_cadence.sh

# 2) noise floor: scrambled rows need a message corpus with NO social relation
#    to the run, e.g. the threads file of any completed E1 run:
export SCRAMBLED_CORPUS="$PWD/simulation_threads_<jobid>_<arrayid>.json"
sbatch bash_job_scripts/v2/mila_v2_noise_floor.sh
```

Jobs must be submitted from the repo root (paths `src/main.py`, `./proportions`,
`news_tweets/` are relative, matching the V1 convention).

Re-running failed array indices: `sbatch --array=<idx1>,<idx2>,... <script>` —
the design row, and therefore the run config and seed, only depends on the index.

## Outputs (per run)

- `survey_v2_<experiment>_<runid>.json` — surveys incl. per-agent dual-order
  answers and log-prob margins (`details`).
- `visualizer_v2_<experiment>_<runid>.json` — full run record (graph, threads,
  surveys, behavioral metrics, resolved `run_parameters` incl. seed and
  empirical graph mean degree).
- `behavioral_metrics_v2_<experiment>_<runid>.json` — echo-chamber + herd metrics.
- `simulation_threads_<jobid>_<arrayid>.json` — raw threads (detector training data).

## Analysis

After (or while) runs complete:

```bash
python analysis/build_dataset.py --input-glob 'visualizer_v2_*.json' --output v2_runs.csv
python analysis/anova_v2.py --dataset v2_runs.csv --outdir analysis_out
python analysis/noise_floor_analysis.py --dataset v2_runs.csv --visualizer-glob 'visualizer_v2_*.json' --outdir analysis_out
python analysis/topology_analysis.py --dataset v2_runs.csv --outdir analysis_out
python analysis/train_bert_detector.py --sim-threads-glob 'simulation_threads_*.json' --human-data <blueprint_threads.json> --outdir detector_out
python analysis/eval_detector.py --model-dir detector_out --sim-threads-glob 'simulation_threads_*.json' --crosscheck-model desklib/ai-text-detector-v1.01
```
